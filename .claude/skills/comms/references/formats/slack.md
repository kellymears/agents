# Format: Slack Messages

## Register Rules

Slack messages exist on a spectrum. Match the register to the situation.

### Quick/casual

- Lowercase is default. Capitalize when the thought is substantial enough to warrant it.
- Sentence fragments are fine. One-word reactions are fine.
- Two-dot ellipses (`..`) for trailing thoughts. Never three dots.
- "lol", "lmao", "haha" deployed naturally — not as punctuation on every message, but when something is genuinely funny or the tone calls for lightness.
- Playful misspellings happen ("guise" for "guys"). Don't force them but don't correct them if the tone is right.
- Exclamation marks only when genuinely excited. One is enough.
- Links dropped inline. No "check out this link:" preamble.

### Work/technical

- Full sentences, proper capitalization.
- State what you already understand before asking a question. This shows you've done the work.
- Reference specific things by name: files, tools, PRs, people (with @mentions).
- Code snippets and commands inline with backticks.
- Keep questions direct and answerable.

### Long-form/vulnerable

- No headers, no bullets, no formatting scaffolding. Just paragraphs.
- Narrative arc through paragraph order, not through structural signposts.
- Vulnerability without performance: state the feeling directly, don't dress it up.
- "freaking" as intensifier. "rad" as genuine vocabulary.
- Builds to emotional crescendo without announcing it.
- Can end with an offer or invitation. Not a summary.

## Structure

- No greetings ("Hey team!") unless the message is genuinely addressed to a group that needs greeting.
- No sign-offs ("Thanks!", "Best,"). This is Slack.
- No subject lines.
- Thread context: if replying in a thread, don't restate what the thread is about.
- For multi-point messages, paragraph breaks are sufficient. Bullets only when the content is genuinely a list (like a set of options or steps).

## Length

Match the content. A one-liner is fine. A multi-paragraph message is fine if the thought requires it. Don't pad short thoughts or compress long ones.
